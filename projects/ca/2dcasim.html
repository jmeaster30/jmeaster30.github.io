2D Cellular Automata Simulator

  • Description

    This program allows you to simulate any 2d cellular automata that you can think of. The freedom of this simulator comes from the ability to embed a lua script with the instructions for your cellular automaton.

  • Usage

    • Running the Simulator

      To build this simulator you need OpenGL and the GLUT libraries. Installing freeglut should be satisfactory. You also need the LUA C/C++ API installed. The download instructions are on the lua website. Once you have all of that installed and you run 'make', you can run one of the examples. So, assuming you cloned the project from git and just ran make, you should be in the directory with the simulator program and the folder named 'automata' should have some example scripts in it.

      To run the Conway's Game of Life cellular automaton you should run the command:

      ./casim conwaysgol.lua

      Once you run that command, a window will pop up and you can then play with the Game of Life CA.

    • Simulator Controls

      There are some controls you have to experiment with your cellular automaton.

      • Arrow Keys: Changes the selected state (the color of the current state is shown with a dot at the mouse position. this also serves to show where the user is pointing on the grid)
      • WASD Keys: Moves the camera around
      • Mouse Scroll: Controls how zoomed in the camera is.
      • Left Click: Sets the cell that the mouse is over to the selected state.
      • C: Sets every cell's state to zero. Essentially clearing the grid.
      • R: Sets every cell to a random state.
      • E: Advances the grid one generation based on the provided rules.
      • Space: Starts and stops the simulation. (A message pops up in the console for when the simulation is running of stopped)

    • Making a Runnable Lua Script

      The barebones runnable lua script has only a "process" function specified below. You can also add in some extra variables that control various things like how many cells are in a grid to the color of the states to how many states there are. I really wanted to make this simulator as open ended as possible. The following is a list of optional variables you can include to change how the simulator runs.

      • win_title = "Cellular Automata" --the title of the window
      • win_width = 640 --the width in pixels of the window
      • win_height = 480 --the height in pixels of the window
      • move_speed = 2.0 --the movement speed of the camera
      • zoom_speed = 5.0 --the zoom speed of the camera
      • cell_size = 20 --the width and height of one cell
      • cells_wide = win_width / cell_size --the width of the grid in cells
      • cells_high = win_height / cell_size --the height of the grid in cells
      • num_of_states = 2 --the number of states each cell can be
      • state_colors = { "FFFFFF", "000000" } --the colors (in hex) of the state (low-high). this array must have at least num_of_states number of elements.
      • kernel_width = 3 --the width of the kernel_width or neighborhood centered on the current cell
      • kernel_height = 3 --the height of the kernel or neighborhood centered on the current cell
      • get_neighbors = false --when false you get a lua array of the number of neighbors of each state. when true you get a 1d lua array of the entire kernel
      • updates_per_second = 60 --this controls how frequently the grid updates

      The only thing required in the lua script is a function called "process" that takes 2 parameters and returns an integer in the range [0, num_of_states). The first parameter is an integer representing the state of the cell currently being processed. If get_neighbors is true then the second parameter will be an array of length kernel_width * kernel_height and contain the values of all the states inside the current kernel. Note: this is a one-dimensional array of two-dimensional data it is organized as such so indexing isn't hard. If get_neighbors is false then the second parameter will be an array of length num_of_states and at each index of the array is the number of neighbors with the state equal to that index (I'll try to think of a better way to say that).   • Examples

    • Conway's Game of Life: the Conway's Game of Life cellular automaton is implemented in the conwaysgol.lua script.

    • Langton's Ant: the Langton's Ant cellular automaton is implemented in the langtonsant.lua script.

    • Wireworld: the Wireworld cellular automaton is implemented in the wireworld.lua script.

    • Brian's Brain: the Brain's Brain cellular automaton is implemented in the briansbrain.lua script.

  • Improvements

    This program can probably be improved in many ways. I could think more about the controls and map them in a more thoughful way. But one of the biggest improvements I can think of is move the processing to the GPU since applying the rules for a cellular automata is highly parallelizable. I think I can implement this improvement by using opengl shaders or some other library that lets me talk to the GPU (CUDA or OpenCL).
